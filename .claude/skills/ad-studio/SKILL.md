# AD STUDIO — AI 광고 제작안 생성기

제품 정보를 입력하면 씬별 광고 제작안(스토리보드·자막·이미지/영상 프롬프트)을 자동 생성하고,
마음에 드는 씬들을 골라 하나로 이어지는 멀티샷 광고 프롬프트까지 만드는 React 아티팩트다.

## 이 스킬이 하는 일

`assets/ad-studio.jsx`에 완성된 아티팩트가 들어 있다. 이 스킬의 핵심 임무는 **그 파일을 그대로 아티팩트로 띄우는 것**이다.
직접 처음부터 다시 코딩하지 말 것 — 이미 검증된(분할 API 호출·노텍스트 규칙·복사 폴백·멀티샷·이미지 분석)
구현이므로, 재작성하면 이 기능들이 누락되고 토큰 잘림·파싱 오류가 재발한다.

## 워크플로우

### 1. 처음 사용하는 사용자에게 — 온보딩 질문

사용자가 광고 소재나 제품 맥락을 아직 주지 않았다면, **아티팩트를 띄우기 전에** 아래를 물어
초기 브리프를 확보한다. 이미 대화에 답이 있으면 다시 묻지 말고 그대로 사용한다.
한 번에 몰아 묻되, 사용자가 답을 건너뛰면 도구 안에서 직접 입력하면 된다고 안내한다.

물어볼 항목:

1. **제품명** (필수) — 예: "루테 루테인 지아잔틴 복합제"
2. **핵심 소구점** — 광고에서 강조할 강점 2~3개. 예: "눈 건강, GMP 인증, 하루 한 알"
3. **광고 포맷** — 셋 중 하나:
   - **프리미엄 부유샷** — 제품이 공중에 떠 있는 럭셔리 매크로 연출
   - **숏폼 광고** — 3초 훅, 빠른 컷, 트렌디한 에너지 (릴스/틱톡)
   - **TV CF** — 시네마틱 브랜드 필름 톤
4. **무드** — 프리셋 10종 중 선택(직접 입력도 가능): 프리미엄/럭셔리, 감성적/따뜻한, 세련된/미니멀,
   강렬한/임팩트, 신뢰감/전문적인, 밝고 산뜻한, 트렌디한 숏폼 스타일, 영화 같은 시네마틱,
   홈쇼핑/판매 집중형, 브랜드 필름 스타일
5. **영상 길이** — 1~15초 (3초당 씬 1개, 최대 5씬으로 자동 분할)
6. **화면 비율** — 16:9(가로) / 9:16(세로) / 1:1(정사각)
7. **제품 이미지 유무** — 실제 제품 사진이 있으면 도구에서 업로드 → AI가 분석해 스타일을 프롬프트에 반영.
   없으면 제품명 기반으로 외형을 프롬프트에서 묘사한다.

질문을 마치면 "지금 알려주신 내용은 도구에 미리 채워둘게요. 나머지는 화면에서 바로 조정하실 수 있어요"라고 안내한 뒤 2단계로 넘어간다.

### 2. 아티팩트 띄우기

`assets/ad-studio.jsx`의 **전체 내용을 그대로** React 아티팩트(`application/vnd.ant.react`)로 출력한다.
파일은 기본값(예: 포맷 'float', 9:16, 6초)으로 시작하도록 되어 있고, 사용자는 UI에서 모든 값을 조정한다.

**선택적 사전 채움**: 사용자가 1단계에서 제품명·포맷 등을 확정했다면, `useState`의 `form` 초기값만
그 값으로 바꿔서 띄워도 좋다(예: `product: "루테 루테인..."`, `format: "shortform"`, `ratio: "16:9"`).
그 외 로직·UI는 절대 변경하지 말 것. 확신이 없으면 원본 그대로 띄우고 사용자가 입력하게 한다.

포맷 id 매핑: 프리미엄 부유샷=`float`, 숏폼 광고=`shortform`, TV CF=`tvcf`.

### 3. 사용법 안내 (아티팩트와 함께)

아티팩트를 띄운 뒤 짧게 사용법을 덧붙인다:

- 왼쪽 브리프 패널에 제품 정보를 채우고 **"제작안 생성"** → 스토리 아크 + 씬별 카드가 생성됨
- 각 씬 카드: 스토리보드 / 자막 카피 / 이미지 프롬프트(한글·영문) / 영상 프롬프트(한글·영문), 모두 개별 복사 가능
- 씬 카드의 **체크박스로 2개 이상 선택** → 하단 MULTI-SHOT에서 하나로 이어지는 멀티샷 프롬프트(4종) 생성
- 상단/하단 **전체 복사** 버튼으로 제작안 전체를 한 번에 복사
- 제품 이미지를 올리면 AI가 분석해 추천 스타일·무드를 제안

## 도구가 지키는 핵심 규칙 (변경 금지)

이 규칙들은 아티팩트 코드에 이미 내장되어 있다. 재작성하거나 축약하지 말 것:

- **노텍스트 규칙**: 모든 한글/영문 프롬프트에 "화면 안에 한글 텍스트, 자막, 로고, UI 텍스트, 워터마크,
  깨진 문자를 생성하지 말 것"(영문: 동일 의미)이 항상 자동 부착된다. AI 생성 프레임에 한글이 깨져 들어가는 것을 막는 핵심 규칙.
- **분할 API 호출**: 씬당 2회(이미지/영상), 멀티샷은 3~4회로 나눠 호출해 토큰 잘림을 방지한다.
- **이미지 우선 파이프라인**: 이미지 프롬프트는 시작 프레임용, 영상 프롬프트는 그 프레임을 움직이는 용도 —
  Higgsfield 등에서 이미지 생성 → 영상 애니메이션 순서에 맞춘다.
- **스토리 연결성**: 씬 생성 전 전체 스토리 아크를 먼저 설계하고, 각 씬은 직전 씬 컨텍스트를 받아 이어지도록 만든다.
- **복사 폴백**: `navigator.clipboard` 실패 시 `execCommand('copy')`로 대체한다.

## 주의

- 아티팩트는 `window.storage`/`localStorage`를 쓰지 않는다(React state만 사용). 그대로 유지할 것.
- 폼 태그(`<form>`) 대신 이벤트 핸들러를 쓴다. 그대로 유지할 것.
- 원본 파일이 1600줄가량으로 길다. 반드시 `view`로 `assets/ad-studio.jsx` 전체를 읽어 빠짐없이 아티팩트로 옮긴다.
